# Plan 08-01: Skeleton Component Library + View Integration

## Objective
Rebuild the Skeleton base component with warm design tokens and add view-specific loading skeletons to all 8 views (Overview, My Tasks, Board, Table, Timeline, Calendar, Settings, Profile). Replace the current spinner-based loading with skeleton placeholders that mirror each view's actual layout.

## Tasks

### Task 1: Rebuild Skeleton.jsx base component library

<read_first>
- frontend/src/components/Skeleton.jsx (current skeleton — 22 lines, uses bg-slate-100)
- frontend/tailwind.config.js (warm color tokens)
- frontend/src/components/Overview.jsx:26-34 (existing HeaderSkeleton pattern using warm tokens)
</read_first>

<action>
Rewrite `frontend/src/components/Skeleton.jsx` to export a component library:
- `Skeleton` — base rectangle placeholder. Props: `className`, `variant` ("primary" | "secondary"). Default: `bg-light-bg-tertiary dark:bg-dark-bg-tertiary animate-pulse rounded-md`. Secondary: `bg-light-bg-hover dark:bg-dark-bg-hover`.
- `SkeletonText` — text line placeholder. Props: `width` (Tailwind class like "w-64"), `height` (default "h-4"). Combines with base Skeleton styling.
- `SkeletonCircle` — circular placeholder. Props: `size` (default "w-12 h-12"). Uses `rounded-full` instead of `rounded-md`.
- `SkeletonCard` — card-shaped placeholder. Props: `className`. Wraps children or shows default block.
- `HeaderSkeleton` — header area (h1 + subtitle). Uses warm tokens matching Overview.jsx pattern: `bg-light-bg-hover dark:bg-dark-bg-hover`.
- All components wrapped in `motion.div` with `initial={{ opacity: 0 }} animate={{ opacity: 1 }} transition={{ duration: 0.3 }}` for smooth appearance.
- Remove all references to `bg-slate-100`.
</action>

<acceptance_criteria>
- `frontend/src/components/Skeleton.jsx` does not contain "slate"
- File exports: `Skeleton`, `SkeletonText`, `SkeletonCircle`, `SkeletonCard`, `HeaderSkeleton`
- All components use `bg-light-bg-tertiary dark:bg-dark-bg-tertiary` or `bg-light-bg-hover dark:bg-dark-bg-hover`
- All components include `animate-pulse`
- HeaderSkeleton uses warm tokens matching `Overview.jsx:26-34` pattern
- `npm run lint` passes
</acceptance_criteria>
