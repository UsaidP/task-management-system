### Task 1: Add route transition wrapper to AppLayout

<read_first>
- frontend/src/components/layout/AppLayout.jsx (full file — contains Outlet for route rendering)
- frontend/src/App.jsx (routing structure)
</read_first>

<action>
In `frontend/src/components/layout/AppLayout.jsx`:

1. Import `AnimatePresence` and `motion` from `framer-motion`, and `useLocation` from `react-router-dom`

2. Wrap the `<Outlet />` component with AnimatePresence for route transitions:
```jsx
const location = useLocation()

// In the render, replace <Outlet /> with:
<AnimatePresence mode="wait">
  <motion.div
    key={location.pathname}
    initial={{ opacity: 0, y: 8 }}
    animate={{ opacity: 1, y: 0 }}
    exit={{ opacity: 0, y: -8 }}
    transition={{ duration: 0.2, ease: "easeOut" }}
    className="flex-1 overflow-hidden"
  >
    <Outlet />
  </motion.div>
</AnimatePresence>
```

3. Ensure the parent container has `relative` positioning so AnimatePresence works correctly.

4. Keep the transition duration short (200ms) to feel snappy, not sluggish.
</action>

- frontend/src/components/ErrorFallback.jsx (already fixed in 08-02)
</read_first>

<action>
In `frontend/src/components/ErrorBoundary.jsx`:

1. Add an `onReset` callback that clears any stale state by forcing a page reload as last resort:
```jsx
const ErrorBoundary = ({ children }) => {
  const handleReset = () => {
    window.location.reload()
  }

  return (
    <ReactErrorBoundary
      FallbackComponent={(props) => <ErrorFallback {...props} onReset={handleReset} />}
      onError={(error, errorInfo) => {
        console.error("ErrorBoundary caught:", error, errorInfo)
      }}
    >
      {children}
    </ReactErrorBoundary>
  )
}
```

2. Pass `onReset` to ErrorFallback so the "Try again" button can call it.
</action>

</acceptance_criteria>
