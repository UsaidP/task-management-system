# Plan 07-03: Table View Rebuild with @tanstack/react-table

**Objective:** Rebuild TableView using @tanstack/react-table with all required columns, sorting, filtering, and design system compliance.

## Context

The existing TableView is a 366-line manual implementation with limited sorting (only title and dueDate), no sprint column, no sprint filter, hardcoded colors, debug styles, and a title that says "List View". This plan rebuilds it using @tanstack/react-table v8 (headless) with full column support.

## Tasks

### Task 1: Install @tanstack/react-table

**read_first:**
- `frontend/package.json` — current dependencies

**action:** Run `npm install @tanstack/react-table` in the `frontend/` directory.

**acceptance_criteria:**
- `@tanstack/react-table` appears in `frontend/package.json` dependencies
- `npm install` completes without errors

### Task 2: Rewrite TableView — Column Definitions + Core Table

**read_first:**
- `frontend/src/components/table/TableView.jsx` — existing code to replace
- `frontend/tailwind.config.js` — design tokens
- `frontend/src/components/context/FilterContext.jsx` — filter API
- `@tanstack/react-table` docs (v8 column helper pattern)

**action:** Rewrite `frontend/src/components/table/TableView.jsx`:

1. **Imports:** `dayjs`, `motion` from framer-motion, `useEffect/useState/useMemo`, `useReactTable`, `getCoreRowModel`, `getSortedRowModel`, `getFilteredRowModel`, `flexRender` from `@tanstack/react-table`, icons, `apiService`, `useAuth`, `useFilter`, `TaskDetailPanel`

2. **Column definitions** using `createColumnHelper()`:
   - **Title:** `accessorKey: 'title'`, cell shows title + description preview
   - **Status:** `accessorFn: (row) => row.status`, cell renders status badge with design tokens
   - **Priority:** `accessorFn: (row) => row.priority`, cell renders priority badge with design tokens
   - **Assignee:** `accessorFn: (row) => row.assignedTo`, cell renders avatar stack
   - **Project:** `accessorFn: (row) => typeof row.project === 'object' ? row.project?.name : 'Personal'`
   - **Sprint:** `accessorFn: (row) => row.sprint?.name || 'Backlog'`
   - **Due Date:** `accessorKey: 'dueDate'`, cell renders formatted date with overdue highlight

3. **Table instance:** `useReactTable({ data: filteredTasks, columns, getCoreRowModel, getSortedRowModel, getFilteredRowModel, state: { sorting }, onSortingChange })`

4. **Header rendering:** Use `flexRender(header.column.columnDef.header, header.getContext())` with sort toggle on click via `header.column.getToggleSortingHandler()`

5. **Row rendering:** Map `table.getRowModel().rows`, each row clickable to open TaskDetailPanel

6. **Fix title:** Change "List View" to "Table"

7. **Remove debug styles:** Delete `style={{ boxShadow: "0px 0px 1px 0.1px #000000" }}`

**acceptance_criteria:**
- `TableView.jsx` imports `useReactTable`, `createColumnHelper`, `flexRender`, `getCoreRowModel`, `getSortedRowModel`, `getFilteredRowModel` from `@tanstack/react-table`
- Table has 7 columns: Title, Status, Priority, Assignee, Project, Sprint, Due Date
- Page title says "Table" (not "List View")
- No hardcoded `slate`, `blue`, `amber`, `emerald` color classes
- No `boxShadow` inline style on root div
- Header shows sort indicators (up/down arrows) when column is sorted

### Task 3: Implement Sorting for All Columns

**read_first:**
- `frontend/src/components/table/TableView.jsx` — rewritten file

**action:** In the rewritten TableView:
1. Add `sorting` state: `const [sorting, setSorting] = useState([])`
2. Pass to table: `state: { sorting }, onSortingChange: setSorting`
3. Enable sorting on all columns via `enableSorting: true` (default) on column definitions
4. Custom sort functions for:
   - Status: `todo < in-progress < under-review < completed` (logical order)
   - Priority: `low < medium < high < urgent` (logical order)
   - Due Date: chronological (nulls last)
   - Project/Sprint: alphabetical
5. Click header to sort asc → desc → none cycle

**acceptance_criteria:**
- Clicking any column header sorts the table
- Second click reverses sort direction
- Sort indicators (FiArrowUp/FiArrowDown) appear on sorted column
- Status sorts in logical workflow order (not alphabetical)
- Priority sorts in severity order (not alphabetical)
- Due date sorts chronologically with nulls at end

### Task 4: Implement Column Filters

**read_first:**
- `frontend/src/components/table/TableView.jsx` — rewritten file
- `frontend/src/components/context/FilterContext.jsx` — sidebar filter API

**action:** In the rewritten TableView:

1. **Sidebar filter integration:** Use `useFilter()` to get `projectFilter` and `sprintFilter`. Apply as pre-filter before table data:
   ```jsx
   const tableData = useMemo(() => {
     let result = tasks
     if (projectFilter) result = result.filter(...)
     if (sprintFilter) result = result.filter(...)
     return result
   }, [tasks, projectFilter, sprintFilter])
   ```

2. **Local column filters:** Add dropdown filters in header for Status, Priority, Project, Sprint:
   - State: `const [columnFilters, setColumnFilters] = useState([])`
   - Pass to table: `state: { sorting, columnFilters }, onColumnFiltersChange: setColumnFilters`
   - Use `column.setFilterValue()` on select change
   - Filter functions: `filterFn: 'equals'` for status/priority, `filterFn: 'includesString'` for project/sprint

3. **Clear filters button:** Show when any filter is active, clears both column filters and sidebar filters

**acceptance_criteria:**
- Status dropdown filters table by status value
- Priority dropdown filters by priority value
- Project dropdown filters by project name
- Sprint dropdown filters by sprint name
- Sidebar project/sprint filter also filters table data
- "Clear filters" button appears when any filter is active
- Clearing filters restores full task list
