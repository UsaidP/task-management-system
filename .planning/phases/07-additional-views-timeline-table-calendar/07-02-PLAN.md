# Plan 07-02: Timeline View Rebuild

**Objective:** Rebuild TimelineView with working zoom, sprint grouping, sidebar filter integration, and design system compliance.

## Context

The existing TimelineView has broken zoom (buttons change state but grid never changes), hardcoded Tailwind colors (`slate-400`, `blue-500`), debug `boxShadow` inline style, no sprint grouping, and ignores sidebar filters. This plan rebuilds it from scratch using the existing code as reference for API patterns.

## Tasks

### Task 1: Rewrite TimelineView — Core Structure + Design Tokens

**read_first:**
- `frontend/src/components/date/TimelineView.jsx` — existing code to replace
- `frontend/tailwind.config.js` — design tokens: `task-status-*`, `task-priority-*`, `accent-*`, `light-*`, `dark-*`
- `frontend/src/components/context/FilterContext.jsx` — `useFilter()` API

**action:** Rewrite `frontend/src/components/date/TimelineView.jsx` with:

1. **Imports:** `dayjs`, `motion` from framer-motion, `useEffect/useState/useMemo`, icons (`FiCalendar`, `FiChevronLeft`, `FiChevronRight`, `FiZoomIn`), `apiService`, `useAuth`, `useFilter`, `TaskDetailPanel`

2. **State:** `tasks`, `loading`, `zoom` (default `"month"`), `selectedTask`, `currentDate`, `groupBy` (default `"project"`)

3. **Filter integration:** After fetching tasks with `apiService.getAllTaskOfUser()`, apply `projectFilter` and `sprintFilter` from `useFilter()`:
   ```jsx
   const { projectFilter, sprintFilter } = useFilter()
   const filteredTasks = useMemo(() => {
     let result = tasks
     if (projectFilter) {
       result = result.filter(t => {
         const pid = typeof t.project === 'object' ? t.project?._id : t.project
         return pid === projectFilter
       })
     }
     if (sprintFilter) {
       result = result.filter(t => t.sprint === sprintFilter)
     }
     return result
   }, [tasks, projectFilter, sprintFilter])
   ```

4. **Remove debug styles:** Delete `style={{ boxShadow: "0px 0px 1px 0.1px #000000" }}` from root div

5. **Design tokens — status colors:**
   - `todo` → `bg-task-status-todo`
   - `in-progress` → `bg-task-status-progress`
   - `under-review` → `bg-task-status-review`
   - `completed` → `bg-task-status-done`

6. **Design tokens — priority colors:**
   - `low` → `border-l-task-priority-low`
   - `medium` → `border-l-task-priority-medium`
   - `high` → `border-l-task-priority-high`
   - `urgent` → `border-l-task-priority-urgent`

**acceptance_criteria:**
- `TimelineView.jsx` contains no hardcoded `slate`, `blue`, `amber`, `emerald` color classes
- Root div has no `boxShadow` inline style
- Component imports `useFilter` from `FilterContext`
- Tasks are filtered by `projectFilter` and `sprintFilter` from filter context
- Status colors use `task-status-todo`, `task-status-progress`, `task-status-review`, `task-status-done`
- Priority colors use `task-priority-low`, `task-priority-medium`, `task-priority-high`, `task-priority-urgent`

### Task 2: Implement Working Zoom (Day/Week/Month)

**read_first:**
- `frontend/src/components/date/TimelineView.jsx` — current broken zoom implementation

**action:** In the rewritten `TimelineView.jsx`, implement zoom that actually changes the grid:

1. Replace `getDatesForZoom()` with `getDateColumns()` that returns different arrays based on `zoom`:
   - **Day zoom:** Return array of 14 individual days starting from `currentDate.startOf('day')`. Each item: `{ label: "DD", sublabel: "ddd", start: dayjs, end: dayjs }`
   - **Week zoom:** Return array of 12 weeks starting from `currentDate.startOf('week')`. Each item: `{ label: "W#", sublabel: "MMM D", start: dayjs, end: dayjs }`
   - **Month zoom:** Return array of days in current month (existing behavior). Each item: `{ label: "DD", sublabel: "ddd", start: dayjs, end: dayjs }`

2. Update `getTaskPosition(task)` to use the column array instead of hardcoded month logic:
   - Calculate which columns the task spans based on `startDate` and `dueDate`
   - Return `{ left: percentage, width: percentage }` based on column count

3. Update navigation buttons:
   - Day zoom: prev/next move by 14 days
   - Week zoom: prev/next move by 12 weeks
   - Month zoom: prev/next move by 1 month (existing)

4. Update date header display based on zoom level

**acceptance_criteria:**
- Clicking "Day" zoom shows 14 individual day columns
- Clicking "Week" zoom shows 12 week columns
- Clicking "Month" zoom shows days in current month
- Task bars reposition correctly for each zoom level
- Navigation (prev/next) adjusts date range appropriate to zoom level
- Today-line indicator works at all zoom levels

### Task 3: Implement Sprint Grouping

**read_first:**
- `frontend/src/components/date/TimelineView.jsx` — current project-only grouping
- `frontend/src/components/sprint/SprintView.jsx` — sprint data patterns

**action:** In the rewritten `TimelineView.jsx`:

1. Add group-by toggle in header (next to zoom controls): "Project" | "Sprint" buttons
2. Implement `groupedTasks` useMemo that branches on `groupBy`:

4. Each group row shows tasks for that group in the timeline grid

**acceptance_criteria:**
- Header has "Project" and "Sprint" toggle buttons
- "Project" mode groups tasks by project name (existing behavior)
- "Sprint" mode groups tasks by sprint name, with unsprinted tasks in "Backlog" group
- Group labels show in left sidebar column (w-48)
- Each group row shows task count

### Task 4: Task Detail Panel Integration

**read_first:**
- `frontend/src/components/date/TimelineView.jsx` — existing TaskDetailPanel usage
- `frontend/src/components/task/TaskDetailPanel.jsx` — panel props interface

**action:** Ensure the rewritten TimelineView includes TaskDetailPanel:
1. `selectedTask` state, set on task bar click
2. Render `<TaskDetailPanel isOpen={!!selectedTask} onClose={() => setSelectedTask(null)} task={selectedTask} members={selectedTask?.assignedTo?.map(u => ({user: u})) || []} onTaskUpdated={(updated) => setTasks(prev => prev.map(t => t._id === updated._id ? updated : t))} />`

**acceptance_criteria:**
- Clicking a task bar opens TaskDetailPanel from the right
- Panel shows correct task data
- Closing panel returns to timeline without state loss
- Task updates in panel reflect in timeline task list
