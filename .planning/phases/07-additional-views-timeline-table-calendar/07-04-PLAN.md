# Plan 07-04: Calendar View Rebuild + Filename Fix

The month view and task chip patterns are good and worth keeping.

### Task 2: Rewrite CalendarView — Month View

**read_first:**
- `frontend/src/components/date/CalenderView.jsx` — existing code to replace
- `frontend/tailwind.config.js` — design tokens
- `frontend/src/components/context/FilterContext.jsx` — filter API

**action:** Create `frontend/src/components/date/CalendarView.jsx` with:

1. **Imports:** `dayjs`, `motion`/`AnimatePresence` from framer-motion, `useEffect/useState/useMemo`, icons (`FiChevronLeft`, `FiChevronRight`, `FiPlus`), `apiService`, `useAuth`, `useFilter`, `TaskDetailPanel`, `CreateTaskModal`

2. **State:** `currentDate`, `tasks`, `selectedTask`, `selectedDate`, `isModalOpen`, `viewMode` ("month"/"week"), `hoveredDay`

3. **Filter integration:** Use `useFilter()` to get `projectFilter` and `sprintFilter`. Apply to task list:
   ```jsx
   const { projectFilter, sprintFilter } = useFilter()
   const filteredTasks = useMemo(() => {
     let result = tasks
     if (projectFilter) result = result.filter(...)
     if (sprintFilter) result = result.filter(t => t.sprint === sprintFilter)
     return result
   }, [tasks, projectFilter, sprintFilter])
   ```
   Use `filteredTasks` everywhere instead of raw `tasks`.

4. **Remove debug styles:** Delete `style={{ boxShadow: "0px 0px 1px 0.1px #000000" }}`

5. **Calendar grid:** Keep existing `calendarDays` useMemo logic (42-cell grid with padding). Use `filteredTasks` in `getTasksForDay()`.

6. **Task chips — design tokens:**
   - Priority tint classes:
     - `urgent` → `bg-task-priority-urgent/20 text-task-priority-urgent`
     - `high` → `bg-task-priority-high/20 text-task-priority-high`
     - `medium` → `bg-task-priority-medium/20 text-task-priority-medium`
     - `low` → `bg-task-priority-low/20 text-task-priority-low`
   - Status border classes:
     - `todo` → `border-l-4 border-l-task-status-todo`
     - `in-progress` → `border-l-4 border-l-task-status-progress`
     - `under-review` → `border-l-4 border-l-task-status-review`
     - `completed` → `border-l-4 border-l-task-status-done`

7. **Overflow handling:** Show first 3 tasks, then "+N more" badge (existing pattern — keep)

8. **Navigation:** Month prev/next, "Today" button (existing — keep)

9. **Header:** "Calendar" title, "View and manage your tasks by date" subtitle

**acceptance_criteria:**
- `CalendarView.jsx` contains no hardcoded `slate`, `blue`, `amber`, `emerald` color classes
- Root div has no `boxShadow` inline style
- Month view shows 7-column grid with tasks plotted by dueDate
- Task chips use `task-priority-*` and `task-status-*` design tokens
- "+N more" shows when day has >3 tasks
- Sidebar project/sprint filters apply to displayed tasks
