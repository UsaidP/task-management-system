# Plan 07-01: Shared Filter Context + Sidebar Wiring

**Objective:** Create shared filter state so sidebar project/sprint selections filter all workspace views.

## Context

Currently the sidebar project list navigates to `/board?project=ID` and the sprint selector is a hardcoded mockup showing "Sprint 4: Q3 Goals". Views ignore sidebar selections. This plan creates a `FilterContext` that all views consume, and wires the sidebar to set filter state instead of navigating.

## Tasks

### Task 1: Create FilterContext

**read_first:**
- `frontend/src/components/context/SidebarContext.jsx` — existing context pattern
- `frontend/src/components/context/customHook.js` — where useAuth is exported

**action:** Create `frontend/src/components/context/FilterContext.jsx` with:
- `FilterProvider` wrapping children with context
- State: `projectFilter` (null or project ID string), `sprintFilter` (null or sprint ID string)
- Functions: `setProjectFilter(id)`, `setSprintFilter(id)`, `clearFilters()`
- Export `useFilter` hook: `const { projectFilter, sprintFilter, setProjectFilter, setSprintFilter, clearFilters } = useFilter()`
- Follow exact same pattern as `SidebarContext.jsx` (createContext, Provider component, useX hook)

**acceptance_criteria:**
- `frontend/src/components/context/FilterContext.jsx` exists
- File exports `FilterProvider` component and `useFilter` hook
- `useFilter` returns `{ projectFilter, sprintFilter, setProjectFilter, setSprintFilter, clearFilters }`
- `projectFilter` defaults to `null`
- `sprintFilter` defaults to `null`

### Task 2: Wire FilterProvider into App

**read_first:**
- `frontend/src/App.jsx` — current provider nesting
- `frontend/src/components/context/FilterContext.jsx` — new file

**action:** In `frontend/src/App.jsx`:
1. Import `FilterProvider` from `./components/context/FilterContext.jsx`
2. Wrap `<FilterProvider>` around `<SidebarProvider>` (FilterProvider outside, SidebarProvider inside) so both sidebar and views can access filters
3. The nesting order should be: `AppThemeProvider > AuthProvider > FilterProvider > SidebarProvider > Routes`

**acceptance_criteria:**
- `App.jsx` imports `FilterProvider` from `FilterContext.jsx`
- `<FilterProvider>` wraps `<SidebarProvider>` in the component tree
- `App.jsx` compiles without errors

### Task 3: Wire Sidebar Project List to Filter

**read_first:**
- `frontend/src/components/layout/Sidebar.jsx` — current project navigation
- `frontend/src/components/context/FilterContext.jsx` — filter API

**action:** In `frontend/src/components/layout/Sidebar.jsx`:
1. Import `useFilter` from `../context/FilterContext`
2. Destructure `{ projectFilter, setProjectFilter }` from `useFilter()`
3. In `renderProjectItem()`, change the `<NavLink to={/board?project=${project._id}}>` to an `<button>` or `<div>` that calls `setProjectFilter(project._id)` on click instead of navigating
4. When a project is selected (projectFilter === project._id), add visual highlight: `bg-accent-primary/10 text-accent-primary`
5. Add a "All Projects" button at the top of the project list that calls `setProjectFilter(null)` — visible when a filter is active
6. Keep the `trackProjectClick` call on click

**acceptance_criteria:**
- Clicking a project in sidebar sets `projectFilter` to that project's ID (does NOT navigate)
- Active filter project has visual highlight class `bg-accent-primary/10`
- "All Projects" option appears and clears filter when clicked
- `trackProjectClick(project._id)` still called on project click

### Task 4: Wire Sidebar Sprint Selector to Filter

**read_first:**
- `frontend/src/components/layout/Sidebar.jsx` — current hardcoded sprint mockup
- `frontend/src/service/apiService.js` — `getSprintsByProject` method
- `frontend/src/components/context/FilterContext.jsx` — filter API

**action:** In `frontend/src/components/layout/Sidebar.jsx`:
1. Import `useFilter` and destructure `{ sprintFilter, setSprintFilter, projectFilter }`
2. Replace the hardcoded "Sprint 4: Q3 Goals" sprint selector with a real dropdown:
   - Add state: `const [sprints, setSprints] = useState([])`
   - Use `useEffect` to fetch sprints via `apiService.getSprintsByProject(projectFilter)` when `projectFilter` changes
   - If no project selected, fetch sprints for all user projects (or show "Select a project first")
   - Render a dropdown listing sprints with `sprint.name`
   - On select, call `setSprintFilter(sprint._id)`
   - Show "All Sprints" option that calls `setSprintFilter(null)`
3. When sprint is selected, highlight it in the dropdown

**acceptance_criteria:**
- Sprint selector shows real sprint data from API (not "Sprint 4: Q3 Goals")
- When no project is selected, sprint selector shows "Select a project first" or is disabled
- Selecting a sprint sets `sprintFilter` to sprint ID
- "All Sprints" option clears the sprint filter
- Sprint list updates when project filter changes
